# musicPlayer

Android music player built for the **Allwinner T3 head unit** (in-car Linux/AOSP). Landscape-locked, USB-focused, libVLC backend.

## Features

- libVLC playback with hardware decode (Allwinner T3) plus emulator-safe fallback
- Filesystem scan across every mounted `StorageVolume` (includes USB drives that AOSP car ROMs miss via MediaStore)
- Room cache for fast cold start, atomic rescan via `@Transaction`
- Audio focus state machine: handles transient loss, ducking, delayed grant
- Resume last session: track + position persisted across reboots
- Up Next preview, shuffle, repeat (off / all / one)
- Procedural per-track album art (no remote calls)
- USB hot-plug rescan via `BroadcastReceiver` (foreground only)
- Foreground media service compliant with Android 14+ FGS rules

## Build

```bash
git clone <repo-url>
cd musicPlayer
echo "sdk.dir=$ANDROID_HOME" > local.properties   # or absolute path to Android SDK
./gradlew assembleDebug
```

Release build also uses the debug keystore (auto-generated by Gradle), so no signing setup is required.

Requirements:
- JDK 17
- Android SDK with platforms 36 and build-tools matching AGP 8.13.2
- Internet access on first build (dependency download)

## Project layout

```
app/src/main/java/com/musicplayer/
  MusicPlayerApp.kt            # @HiltAndroidApp entry
  data/
    local/db/                  # Room: SongEntity, SongDao, MusicDatabase
    local/mediastore/          # Filesystem scan, FNV-1a 64-bit IDs
    prefs/                     # PlayerPreferences (DataStore), ScanPreferences (SP)
    repository/SongRepository  # @Singleton concrete class, no interface
  di/DatabaseModule.kt         # Provides Room db
  domain/model/Song.kt
  presentation/
    MainActivity.kt            # Single Activity wires everything
    player/                    # PlayerViewModel, adapters, AlbumArtGenerator
  receiver/UsbMountReceiver.kt
  service/VlcService.kt        # Foreground mediaPlayback service
```

## Permissions

- `READ_MEDIA_AUDIO` (Tiramisu+) / `READ_EXTERNAL_STORAGE` (older)
- `FOREGROUND_SERVICE` + `FOREGROUND_SERVICE_MEDIA_PLAYBACK`

## Known gaps

- Notification has no `MediaSession` / `MediaStyle` controls (lockscreen + Bluetooth/AA metadata not exposed)
- Service stays foreground after pause (no idle `stopForeground`)
- `fallbackToDestructiveMigration()` no-arg deprecation warning (Room 2.8) — cache-only DB so destructive migration is intentional
- Settings sheet rebuilds Button views on each open
